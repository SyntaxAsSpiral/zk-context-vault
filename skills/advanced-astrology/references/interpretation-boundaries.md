# Interpretation Boundaries

Agent astrology work must remain grounded, bounded, and non-deterministic.

## Computed Data vs Interpretation

Keep these categories separate:

- **Computed data:** values produced by scripts or backends, such as planetary hour ruler, tattwa, sunrise, sunset, biorhythm percentages, moon sign, and timestamp.
- **Interpretation:** symbolic synthesis that frames those values as qualitative timing or reflective prompts.

A good response says, "The computed pulse shows X; symbolically, that can be read as Y." It does not say, "X means Y will happen."

## Never Fabricate

Do not invent:

- planetary positions
- moon signs
- sunrise/sunset times
- birth data
- house placements
- aspects
- current location
- biorhythm values

If a signal is unavailable, say so and proceed with the available signals.

## Avoid High-Stakes Claims

Do not use astrology outputs for:

- medical diagnosis or treatment
- financial trades or investment decisions
- legal decisions
- physical safety decisions
- relationship ultimatums
- claims of fate, destiny, certainty, or external command

Prefer low-risk language:

- "a useful posture may be..."
- "this supports reflection around..."
- "consider treating this as..."
- "symbolically, this leans toward..."

## Good Reading Pattern

1. Quote the actual data.
2. State limitations or warnings.
3. Offer a symbolic synthesis.
4. Translate synthesis into practical posture.
5. Invite empirical tracking: "notice whether this actually correlates."

## Bad Reading Pattern

Avoid:

- "You must..."
- "The planets guarantee..."
- "This is a bad day for..."
- "Your body needs..." when unsupported
- "Do not make this decision..." based only on astrology

## Empirical Orientation

Astrology readings in agent contexts are best treated as a structured reflective interface: a way to notice timing, mood, pacing, and symbolic resonance. Encourage users to track what correlates rather than believing every interpretation by default.
